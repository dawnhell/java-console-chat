Консольный многопользовательский чат.

Чат будет работать по принципу клиент/сервер. Т.е. в программе будут два режима	работы - пользовательский и серверный.
Программа будет спрашивать пользователя, в каком режиме ее запустить(либо можно сделать две разные программы: одну под сервер, другую -
под клиентскую часть). 

[DD]: два режима работы -- это коцептуально неверно. чат должен быть из двух программ. одна -- сервер. это просто консольная программа, которую ты запускаешь в терманале, и весь вывод (логи) тоже туда идут.
вторая программа -- клиент. UI приложение. но в первом приближении, для программирования логики это может быть тоже консольное приложение.


По архитектуре - примерно следующее:
	- сервер будет один, а пользователей – любое количество;
	- каждый пользователь подключается к этому серверу и посылает ему свои сообщения;
	- сервер хранит список подключенных пользователей и рассылает им все полученные сообщения;
	- в дальнейшем можно добавить режим переписки с отдельными пользователями + отправку файлов;
